Servidor WoW KODO — Guía para Windows 10

i

Antes de empezar — lee esto

¿Qué vas a montar?

Un servidor WoW que corre en tu propio PC. Tu cliente WoW se conectará a 127.0.0.1 (que significa "yo mismo") en lugar de a Blizzard.

¿Cuánto tarda?

PasoTiempo
Instalar los 6 programas necesarios~1h (mayoría es descarga)
Compilar el servidor~40-50 min (el PC trabaja solo)
Extraer mapas del cliente WoW~1-2h (esperando)
Configurar y arrancar~30 min

¿Qué necesita tu PC?

  • Windows 10 de 64 bits
  • Al menos 8 GB de RAM
  • Al menos 80 GB de espacio libre en el disco E:
ii

Estructura de carpetas — créala ahora antes de hacer nada

Abre el Explorador de Windows y crea esta estructura exacta en tu disco E:

E:\Servidores\KODO\
│
├── source\        ← aquí irá el código de TrinityCore que descargaremos
├── build\         ← archivos temporales de compilación (se pueden borrar después)
├── server\        ← el servidor final compilado
│   ├── bin\       ← los ejecutables (.exe del servidor)
│   ├── data\      ← mapas y datos extraídos del cliente WoW
│   ├── logs\      ← registros del servidor (para ver errores)
│   └── etc\       ← archivos de configuración del servidor
└── backups\       ← copias de seguridad de la base de datos

Para crearlas todas de golpe: abre el menú Inicio, escribe cmd, haz clic derecho → Ejecutar como administrador, y pega esto:

cmd 
mkdir E:\Servidores\KODO\source
mkdir E:\Servidores\KODO\build
mkdir E:\Servidores\KODO\server\bin
mkdir E:\Servidores\KODO\server\data
mkdir E:\Servidores\KODO\server\logs
mkdir E:\Servidores\KODO\server\etc
mkdir E:\Servidores\KODO\backups

Pulsa Enter. Todas las carpetas quedan creadas.

Programas necesarios
1
Programa 1

Visual Studio Community (2022 o 2026)

🛠️

¿Para qué sirve? Es el "constructor" del servidor. Sin él no se puede compilar TrinityCore en Windows.

⚠️

Este es el paso más largo. Descarga e instala varios GB. Ten paciencia.

✅

Si ya tienes Visual Studio Community 2026 instalado (versión 18.x), no tienes que instalar nada. Salta directamente al apartado "Instalar la carga de trabajo" más abajo.

Descargar (solo si NO tienes ningún Visual Studio):

Ve a → https://visualstudio.microsoft.com/es/vs/community/
Haz clic en el botón azul "Descargar Visual Studio". Se descarga un pequeño instalador (~5 MB).

Instalar la carga de trabajo:

Tanto si acabas de instalarlo como si ya lo tenías, necesitas asegurarte de que tiene el componente de C++:

  1. Abre el Visual Studio Installer (búscalo en el menú Inicio).
  2. Junto a tu versión de Visual Studio, haz clic en "Modificar".
  3. Aparece la pantalla de "Cargas de trabajo".
  4. Busca y marca ÚNICAMENTE este recuadro:
    • ✅ "Desarrollo para el escritorio con C++"
  5. Si tienes Visual Studio 2026: ve también a la pestaña "Componentes individuales" (arriba de la ventana), escribe msvc v143 en el buscador, y marca la que diga algo como:
    • ✅ "Herramientas de compilación de MSVC v143 - VS 2022 C++ x64/x86" (cualquier número de versión vale, ej. 14.44-17.14)
    ⚠️

    Cuidado con cuál marcas: en el buscador van a salirte muchísimas variantes casi idénticas (ARM, ARM64, ARM64EC, bibliotecas con mitigación Spectre...). Tu PC casi seguro es x64 normal (Intel/AMD), así que ignora todo lo que diga "ARM" y marca solo la que diga "x64/x86". No importa si pone "(sin soporte técnico)" al lado — sigue siendo instalable y funciona igual.

    ⚠️

    Por qué hace falta esto: VS2026 trae de serie un compilador más nuevo (v145), pero los binarios precompilados de Boost que vas a instalar en el PROGRAMA 5 sólo existen para el compilador v143 (el de VS2022). Microsoft incluye ese compilador "antiguo" como componente opcional dentro del propio instalador de VS2026 justo para estos casos. Si no lo marcas, Boost no encajará más adelante.

  6. No marques nada más.
  7. Haz clic en "Modificar" abajo a la derecha y espera a que instale.
¿Funcionó?

En el menú Inicio aparece Visual Studio Community. No hace falta abrirlo ahora.

Programa 2

Git

🛠️

¿Para qué sirve? Para descargar el código del servidor desde internet.

Descargar:

Ve a → https://git-scm.com/download/win
Se descarga automáticamente un archivo .exe.

Instalar:

  1. Ejecuta el .exe.
  2. Haz clic en Next en todas las pantallas hasta llegar a "Adjusting your PATH environment".
  3. En esa pantalla asegúrate de que está marcado:
    • ✅ "Git from the command line and also from 3rd-party software"
  4. Sigue haciendo clic en Next → Install.
  5. Al terminar → Finish.
¿Funcionó?

Cierra la consola si la tenías abierta, abre una nueva (cmd), escribe git --version y pulsa Enter. Debe aparecer algo como git version 2.x.x.

Programa 3

CMake

🛠️

¿Para qué sirve? Es un intermediario que prepara el código de TrinityCore para que Visual Studio pueda construirlo.

Descargar:

Ve a → https://cmake.org/download/
Busca la sección "Latest Release" y descarga:

  • cmake-X.XX.X-windows-x86_64.msi (el archivo .msi de 64 bits)

Instalar:

  1. Ejecuta el .msi.
  2. Acepta la licencia → Next.
  3. En "Install Options" selecciona:
    • ✅ "Add CMake to the system PATH for all users"
    ⚠️

    Este punto es obligatorio. Si no lo marcas, CMake no funcionará.

  4. Haz clic en Next → Install.
¿Funcionó?

Cierra y abre una consola nueva. Escribe cmake --version. Debe aparecer cmake version 3.x.x.

Programa 4

OpenSSL

🛠️

¿Para qué sirve? Es una librería de seguridad y cifrado que el servidor necesita para funcionar.

Descargar:

Ve a → https://slproweb.com/products/Win32OpenSSL.html

Busca en la lista la fila que diga exactamente:

  • Win64 OpenSSL v3.x.x → descarga el .exe de esa fila
⚠️

Importante: descarga la versión Win64 (no Win32) y la que no pone "Light".

Instalar:

  1. Ejecuta el .exe.
  2. Acepta la licencia → Next.
  3. En "Select destination location" deja la ruta por defecto: 
    C:\OpenSSL-Win64\
    No la cambies — la necesitarás exactamente así después.
  4. En "Select Additional Tasks" marca:
    • ✅ "The OpenSSL binaries (/bin) directory"
  5. Haz clic en Next → Install.
5
Programa 5

Boost

🛠️

¿Para qué sirve? Otra librería que TrinityCore necesita. Viene en versión precompilada para Windows, no hay que hacer nada especial.

⚠️

Importante si tienes Visual Studio 2026: vas a buscar un archivo con msvc-14.4-64 en el nombre y no existe, ni va a aparecer en ninguna carpeta. Microsoft saltó directamente del compilador v143 (el de VS2022) al v145 en VS2026, sin pasar por un "v144" público — por eso Boost nunca ha compilado nada para ese nombre. Lo correcto, tengas VS2022 o VS2026, es descargar siempre el archivo msvc-14.3-64 (ver más abajo). Funciona igual en ambos porque en el PROGRAMA 1 ya instalaste el componente MSVC v143 dentro de tu VS2026.

Descargar:

Ve a → https://sourceforge.net/projects/boost/files/boost-binaries/

  1. Haz clic en la carpeta con el número más alto (ej: 1.87.0).
  2. Dentro, busca y descarga el archivo con msvc-14.3-64 en el nombre — es el mismo para VS2022 y para VS2026: 
    boost_1_87_0-msvc-14.3-64.exe

    No existe ninguna versión "msvc-14.4" ni "msvc-14.5". El msvc-14.3-64 es el correcto sea cual sea tu Visual Studio.

  3. Descárgalo. Puede pesar 500 MB - 1 GB.

Instalar:

  1. Ejecuta el .exe.
  2. Cuando pregunte dónde instalarlo deja la ruta por defecto. Será algo como: 
    C:\local\boost_1_87_0\
    Apunta exactamente esta ruta — la necesitarás en el paso de compilación.
  3. Haz clic en Install y espera.

Programa 6

Base de datos: MariaDB + HeidiSQL

🛠️

¿Para qué sirve? Es la base de datos. Aquí se guarda todo: las cuentas, los personajes, el mundo del juego, las fases...

Vamos a usar MariaDB 10.11 LTS: es ligera, rápida, y su instalador trae integrado HeidiSQL, el programa gráfico que vas a usar durante toda la guía para crear bases de datos y ejecutar consultas SQL.

Descargar:

Ve a → https://mariadb.org/download/

  1. En "Operating System" selecciona Windows.
  2. Descarga el archivo .msi de 64 bits más reciente de la versión 10.11 (es la LTS, la más estable). Se llama algo como mariadb-10.11.X-winx64.msi.

Instalar:

  1. Ejecuta el .msi.
  2. Haz clic en Next → acepta la licencia → Next.
  3. En "Custom Setup" deja todo por defecto → Next.
  4. Pon una contraseña para el usuario root. Usa algo simple: 
    root1234
    Apúntala. La necesitarás siempre.
    • Deja marcado ✅ "Enable access from remote machines for 'root' user" (útil para herramientas gráficas)
  5. Haz clic en Next → Install → Finish.

El instalador de MariaDB incluye HeidiSQL — es el programa gráfico que vas a usar para gestionar la base de datos en el resto de esta guía.

¿Funcionó?

Abre HeidiSQL desde el menú Inicio. En la pantalla de conexión haz clic en "Abrir" — si se conecta sin errores → correcto.

Rutas para CMake (las necesitarás en el PASO 2):

MYSQL_INCLUDE_DIR = C:/Program Files/MariaDB 10.11/include/mysql
MYSQL_LIBRARY     = C:/Program Files/MariaDB 10.11/lib/libmariadb.lib
⚠️

Ajusta 10.11 por la versión exacta que instalaste si es diferente.

Instalación del servidor
1
Paso 1

Descargar el código de TrinityCore

Ahora vamos a descargar el código fuente del servidor WoW.

Abre la consola (menú Inicio → cmd → Enter) y escribe estos dos comandos, uno por uno:

cmd
E:
cmd
cd E:\Servidores\KODO
cmd
git clone -b master https://github.com/TrinityCore/TrinityCore.git source

Esto descarga el código dentro de E:\Servidores\KODO\source\. Verás texto avanzando. Puede tardar 3-8 minutos según tu conexión.

¿Terminó bien?

Al final aparece Resolving deltas: 100% y vuelve el cursor sin mensajes de error.

Consulta siempre qué versión del cliente WoW soporta el master en este momento: https://trinitycore.info/en/home

2
Paso 2

Preparar la compilación con CMake

CMake tiene una interfaz gráfica que facilita este proceso.

Abrir CMake GUI:

Abre el menú Inicio, busca "CMake (cmake-gui)" y ábrelo.

Configurar las rutas:

Verás dos campos en la parte superior. Rellena así (puedes escribir directamente o usar el botón "Browse"):

  • "Where is the source code" → 
    E:/Servidores/KODO/source
  • "Where to build the binaries" → 
    E:/Servidores/KODO/build
⚠️

Usa siempre barras hacia adelante / dentro de CMake, nunca \.

Primera configuración:

  1. Haz clic en el botón "Configure" (abajo a la izquierda).
  2. Aparece una ventana emergente. En el desplegable superior tienes que elegir tu versión de Visual Studio:
    • Si tienes Visual Studio 2022 → selecciona: "Visual Studio 17 2022"
    • Si tienes Visual Studio 2026 (tu caso) → selecciona: "Visual Studio 18 2026"

      Esta opción solo aparece con CMake 3.30 / 4.2 o más reciente. Si no la ves en la lista, actualiza CMake (PROGRAMA 3) a la última versión.

    En el campo "Optional platform for generator" escribe:

    x64

    Justo debajo, en "Optional toolset to use", escribe:

    v143
    ⚠️

    Esto hay que rellenarlo aunque hayas elegido el generador "Visual Studio 18 2026". El motivo: los binarios de Boost que instalaste en el PROGRAMA 5 están compilados con el toolset v143, no con el v145 que trae VS2026 por defecto. Si dejas este campo vacío, CMake usará v145 y es probable que Boost no se detecte bien o falle el enlazado más adelante.

    Deja marcado "Use default native compilers" y haz clic en Finish.

  3. CMake trabaja un momento. La lista central se llena de líneas en rojo — eso es completamente normal.

Ajustar variables:

Ahora busca y modifica estas variables en la lista. Usa el campo de búsqueda arriba de la lista:

Busca CMAKE_INSTALL_PREFIX → haz doble clic en el valor → escribe:

E:/Servidores/KODO/server

Busca TOOLS → en el valor haz clic en la casilla para que quede en ON (marcado).

Busca BOOST_ROOT → haz doble clic en el valor → escribe la ruta donde instalaste Boost. Ejemplo:

C:/local/boost_1_87_0

(ajusta el número de versión al que instalaste tú)

💡

Si al buscar BOOST_ROOT no aparece ninguna fila (puede pasar en la primera Configure), no existe todavía como variable y no puedes "editarla" sin más. En ese caso pulsa el botón "Add Entry" (arriba de la lista) y créala a mano: Name → BOOST_ROOT, Type → PATH, Value → la ruta donde instalaste Boost. Acepta y sigue con el Configure de abajo.

  1. Haz clic en "Configure" otra vez. Las líneas rojas deberían desaparecer casi todas.
💡

Es normal que se queden algunas filas en rojo aunque todo haya ido bien — por ejemplo LIB_EAY_DEBUG, LIB_EAY_RELEASE, SSL_EAY_DEBUG o SSL_EAY_RELEASE (variables internas que crea OpenSSL). El rojo en CMake GUI solo significa "esta variable es nueva o cambió en esta pasada", no significa error. Lo que importa es el texto del log de abajo: si termina en Configuring done sin líneas de error, vas bien y puedes seguir a Generate sin tocar esas filas.

Si siguen quedando líneas rojas con "OPENSSL" en el nombre:

Busca OPENSSL_ROOT_DIR → pon:

C:/OpenSSL-Win64

→ clic en Configure de nuevo.

Si quedan líneas rojas con "MYSQL" en el nombre:

Busca y rellena:

  • MYSQL_INCLUDE_DIR → 
    C:/Program Files/MariaDB 10.11/include/mysql
  • MYSQL_LIBRARY → 
    C:/Program Files/MariaDB 10.11/lib/libmariadb.lib

→ clic en Configure de nuevo.

Cuando no queden líneas rojas:

Haz clic en "Generate". Espera unos segundos hasta ver Generating done abajo.

Haz clic en "Open Project" → se abre Visual Studio.

3
Paso 3

Compilar con Visual Studio

Visual Studio se abre con el proyecto de TrinityCore cargado.

  1. Arriba del todo hay un desplegable que seguramente dice Debug. Cámbialo a: RelWithDebInfo
  2. En el menú superior ve a Compilar → Compilar solución (o pulsa Ctrl+Mayús+B).
  3. Empieza la compilación. Abajo verás un número avanzando: 1/500, 25/500... Esto tarda entre 30 y 60 minutos. El ventilador del PC va a sonar — es normal. No cierres Visual Studio.
  4. Al terminar verás en la barra inferior: 
    ========== Compilación: X correctos, 0 errores ==========
    Si hay 0 errores → perfecto. Si hay errores en rojo → ve a la sección de problemas al final de esta guía.
  5. Instalar los archivos en su carpeta final: En el panel "Explorador de soluciones" (normalmente a la derecha), busca el proyecto que se llama "INSTALL". Haz clic derecho sobre él → Compilar.

    Esto copia todos los archivos del servidor a E:\Servidores\KODO\server\.

¿Funcionó?

Abre E:\Servidores\KODO\server\bin\ en el Explorador. Deberías ver:

  • worldserver.exe
  • bnetserver.exe
  • mapextractor.exe
  • vmap4extractor.exe
  • vmap4assembler.exe
  • mmaps_generator.exe
  • connection_patcher.exe

Si están todos → la compilación fue correcta.

4
Paso 4

Crear las bases de datos

Abre HeidiSQL (búscalo en el menú Inicio).

Haz doble clic en tu sesión guardada para conectar (la contraseña ya debería estar puesta; si te la pide, escribe root1234 o la que pusiste).

A la derecha tienes por defecto una pestaña "Consulta" abierta. Si no la ves, créala con Ctrl+T.

Copia y pega todo el bloque siguiente en esa pestaña:

sql 
-- Crear usuario del servidor
CREATE USER IF NOT EXISTS 'tc'@'localhost' IDENTIFIED BY 'tcpassword';

-- Crear las 5 bases de datos
CREATE DATABASE IF NOT EXISTS `auth`
  CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

CREATE DATABASE IF NOT EXISTS `characters`
  CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

CREATE DATABASE IF NOT EXISTS `world`
  CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

CREATE DATABASE IF NOT EXISTS `hotfixes`
  CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

CREATE DATABASE IF NOT EXISTS `custom`
  CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- Dar permisos al usuario 'tc' sobre todas ellas
GRANT ALL PRIVILEGES ON `auth`.*       TO 'tc'@'localhost';
GRANT ALL PRIVILEGES ON `characters`.* TO 'tc'@'localhost';
GRANT ALL PRIVILEGES ON `world`.*      TO 'tc'@'localhost';
GRANT ALL PRIVILEGES ON `hotfixes`.*   TO 'tc'@'localhost';
GRANT ALL PRIVILEGES ON `custom`.*     TO 'tc'@'localhost';

FLUSH PRIVILEGES;

Pulsa F9 (o haz clic en el botón azul de "Play" de la barra de herramientas) para ejecutarlo todo de golpe.

Abajo, en el panel de resultados, verás líneas confirmando cada sentencia si todo fue bien. Si hay algo en rojo, lee el mensaje de error.

5
Paso 5

Descargar la base de datos del mundo (TDB)

TrinityCore necesita una base de datos con todo el contenido del juego (zonas, criaturas, objetos...) que se descarga por separado.

  1. Ve con tu navegador a: https://github.com/TrinityCore/TrinityCore/releases
  2. Busca la release más reciente que tenga archivos con "TDB" en el nombre.
  3. Descarga el archivo .7z — se llama algo como: 
    TDB_full_world_XXXXXXXX_XX.7z
    Guárdalo en E:\Servidores\KODO\backups\
  4. Si no tienes 7-Zip instalado, descárgalo gratis en https://www.7-zip.org/ e instálalo.
  5. Haz clic derecho sobre el .7z → 7-Zip → Extraer aquí. Obtendrás un archivo .sql enorme.
  6. Importarlo con HeidiSQL:
    • Abre HeidiSQL y conéctate a tu sesión.
    • En el árbol de la izquierda, haz clic en la base de datos world para seleccionarla (debe quedar resaltada en azul).
    • Ve al menú Archivo → "Run SQL file" (puede aparecer como "Ejecutar archivo SQL" según el idioma).
    • Haz clic en los tres puntos ... y busca el archivo .sql que extrajiste.
    • En el desplegable de codificación, selecciona "UTF-8" — no dejes "Auto detect".
    • Haz clic en OK.

    Esto tarda entre 10 y 30 minutos. Verás las líneas ejecutándose y una barra de progreso.

6
Paso 6

Extraer los datos del cliente WoW

Necesitas tu cliente WoW 12.x (el que usas para Epsilon u otro servidor).

Los extractores leen los archivos del cliente WoW y generan los datos de mapas que el servidor necesita.

Copiar los extractores al cliente WoW:

Abre E:\Servidores\KODO\server\bin\ en el Explorador y copia estos 3 archivos a la carpeta donde tienes el cliente WoW:

  • mapextractor.exe
  • vmap4extractor.exe
  • vmap4assembler.exe

Supongamos que tu WoW está en D:\WoW\ — ajusta la ruta a la tuya.

Ejecutar los extractores:

Abre una consola (cmd) y escribe:

cmd
D:
cd D:\WoW

(ajusta a tu ruta real)

Ejecuta en orden:

cmd
mapextractor.exe

Extrae los mapas y datos del juego. Tarda 15-30 minutos. Crea las carpetas dbc, maps, cameras, gt.

cmd
vmap4extractor.exe

Extrae la geometría de colisión. Tarda 1-2 horas. Crea la carpeta Buildings.

cmd
mkdir vmaps
vmap4assembler.exe Buildings vmaps

Monta los vmaps. Tarda 30-60 minutos más. Crea la carpeta vmaps.

Los mmaps (pathfinding de NPCs) tardan 4-8 horas. Para empezar puedes saltarlos — los NPCs no caminarán con inteligencia pero para un servidor RP puro no importa nada. Si los quieres después:

cmd
mkdir mmaps
mmaps_generator.exe

Mover los datos al servidor:

Abre el Explorador y mueve/copia estas carpetas desde D:\WoW\ a E:\Servidores\KODO\server\data\:

  • dbc
  • maps
  • cameras
  • gt
  • vmaps

Así debe quedar tu carpeta del servidor al terminar este paso:

server\
├── bin\                  ← del Paso 3 (INSTALL)
│   ├── worldserver.exe
│   ├── bnetserver.exe
│   ├── mapextractor.exe
│   ├── vmap4extractor.exe
│   ├── vmap4assembler.exe
│   ├── mmaps_generator.exe
│   ├── connection_patcher.exe
│   └── (varias .dll necesarias para arrancar)
│
├── data\                 ← AQUÍ aterriza todo lo de este paso
│   ├── dbc\
│   ├── maps\
│   ├── cameras\
│   ├── gt\
│   ├── vmaps\
│   └── (mmaps\ — opcional, normal que no la tengas aún)
│
├── logs\                 ← vacía todavía, se llena en el Paso 9
└── etc\
    ├── worldserver.conf.dist
    └── bnetserver.conf.dist
        (worldserver.conf y bnetserver.conf no existen aún, se crean en el Paso 7)

Y así debe quedar la carpeta de tu cliente WoW justo después de extraer:

D:\WoW\
├── Wow.exe                    ← el cliente, no lo tocas
├── mapextractor.exe            ← copiado por ti, se puede borrar luego
├── vmap4extractor.exe           ← idem
├── vmap4assembler.exe           ← idem
├── dbc\          ──┐
├── maps\            │
├── cameras\          ├── estas 5 son las que MUEVES a server\data\
├── gt\               │
├── vmaps\         ──┘
└── Buildings\                 ← desechable, bórrala, no se mueve
💡

Solo se mueven dbc, maps, cameras, gt y vmaps. La carpeta Buildings\ del cliente es basura temporal que usó vmap4assembler y ya no sirve para nada — puedes borrarla con confianza.

7
Paso 7

Configurar el servidor

7a — Crear los archivos de configuración

Ve a E:\Servidores\KODO\server\etc\ en el Explorador.

Verás dos archivos que terminan en .dist. Haz dos copias:

  • Copia worldserver.conf.dist → renombra la copia a worldserver.conf
  • Copia bnetserver.conf.dist → renombra la copia a bnetserver.conf

7b — Editar worldserver.conf

Haz clic derecho en worldserver.conf → Abrir con → Bloc de notas.

Usa Ctrl+H (Buscar y reemplazar) para encontrar y cambiar cada línea. Busca la parte izquierda y reemplaza por la línea completa:

Busca esta línea:

LoginDatabaseInfo =

Y deja esa línea como:

7d — Generar los certificados SSL

El servidor de login necesita unos archivos de seguridad. Vamos a generarlos.

Abre la consola (cmd) y ejecuta:

cmd 
cd E:\Servidores\KODO\server\etc
"C:\OpenSSL-Win64\bin\openssl.exe" req -x509 -newkey rsa:2048 -keyout bnetserver.key.pem -out bnetserver.cert.pem -days 3650 -nodes -subj "/CN=127.0.0.1"

Se crean dos archivos .pem en la carpeta etc. El servidor los usa para el cifrado del login — no hace falta entender qué son.

Paso 9

Primer arranque

Arrancar los servidores por primera vez

  1. Haz doble clic en 1_arrancar_bnetserver.bat. Se abre una ventana verde. Espera hasta ver esta línea: 
    Started realm list socket listening on 0.0.0.0:1119
  2. Sin cerrar esa ventana, haz doble clic en 2_arrancar_worldserver.bat. Se abre una ventana azul.

    La primera vez tarda varios minutos — está creando todas las tablas de la base de datos automáticamente. Verás texto desfilando. Espera hasta ver:

    World initialized in X minutes

Crear tu cuenta de GM

Cuando aparezca World initialized, la ventana azul del worldserver tiene una consola donde puedes escribir. Haz clic en esa ventana y escribe (ajusta a tu gusto):

account create admin@kodo micontraseña
account set gmlevel admin@kodo 3 -1

El 3 -1 significa: nivel GM 3 (máximo) en todos los reinos (-1).

Configurar el realm en la base de datos

Vuelve a HeidiSQL, abre una nueva pestaña de consulta (Ctrl+T) y ejecuta:

sql 
USE auth;

UPDATE realmlist
SET address          = '127.0.0.1',
    localAddress     = '127.0.0.1',
    localSubnetMask  = '255.255.255.0',
    port             = 8085,
    name             = 'KODO'
WHERE id = 1;

Pulsa F9 (o el botón de Play) para ejecutarlo.

💡

¿Por qué basta con el USE auth; de arriba? Al pulsar F9, HeidiSQL ejecuta todo el contenido de la pestaña de arriba a abajo, en orden, sobre la misma conexión. La línea USE auth; cambia la base de datos activa justo antes de que se ejecute el UPDATE — no necesitas además hacer clic en auth en el árbol de la izquierda, el cambio de base de datos ya va incluido en la propia consulta.

Paso 10

Conectar el cliente WoW con Arctium Game Launcher

El cliente WoW está configurado para conectarse a Blizzard. Tenemos que redirigirlo a tu servidor. Para esto vamos a usar Arctium Game Launcher: un lanzador oficial que conecta el cliente a servidores personalizados sin modificar ni generar ningún .exe parcheado.

Descargar Arctium Game Launcher:

Ve a → https://arctium.io/wow/
Busca la tarjeta "Custom Server Game Launcher" y haz clic en "Download Launcher". Se descarga un archivo .zip.

Instalar el lanzador:

  1. Extrae el contenido del .zip descargado.
  2. Copia el archivo Arctium Game Launcher.exe que hay dentro a la carpeta de tu cliente WoW — la misma carpeta donde está Wow.exe (ej: D:\WoW\).

Configurar el portal:

Dentro de la carpeta del WoW, entra en la carpeta WTF y abre el archivo Config.wtf con el Bloc de notas. Añade esta línea al final:

SET portal "127.0.0.1"

Guarda el archivo.

✅

Arctium Game Launcher exige que el servidor de login tenga un certificado TLS válido cuyo nombre coincida con la dirección del portal. En tu caso ya está cubierto: el certificado que generaste en el PASO 7d se creó con -subj "/CN=127.0.0.1", exactamente la misma dirección que acabas de poner en Config.wtf y en el realmlist. No tienes que hacer nada extra.

Paso 12

Tablas SQL del sistema de fases personalizado

Con el servidor funcionando, crea la estructura de la base de datos para el sistema de fases. En HeidiSQL, abre una nueva pestaña de consulta (Ctrl+T) y ejecuta:

sql 
USE custom;

-- Tabla principal: cada fase tiene un nombre, dueño y descripción
CREATE TABLE IF NOT EXISTS `custom_phases` (
  `phase_id`    INT UNSIGNED NOT NULL AUTO_INCREMENT,
  `owner_guid`  BIGINT UNSIGNED NOT NULL,
  `name`        VARCHAR(64) NOT NULL,
  `description` TEXT,
  `created_at`  TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (`phase_id`),
  KEY `idx_owner` (`owner_guid`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- Tabla de miembros: qué jugadores están en cada fase
CREATE TABLE IF NOT EXISTS `custom_phase_members` (
  `phase_id`    INT UNSIGNED NOT NULL,
  `player_guid` BIGINT UNSIGNED NOT NULL,
  `permissions` INT UNSIGNED NOT NULL DEFAULT 0,
  -- 0 = solo ver  |  1 = puede construir  |  2 = co-propietario
  PRIMARY KEY (`phase_id`, `player_guid`),
  CONSTRAINT `fk_pm_phase` FOREIGN KEY (`phase_id`)
    REFERENCES `custom_phases`(`phase_id`) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- Tabla de objetos del juego (gobjects) dentro de cada fase
CREATE TABLE IF NOT EXISTS `custom_phase_gameobjects` (
  `go_id`     BIGINT UNSIGNED NOT NULL AUTO_INCREMENT,
  `phase_id`  INT UNSIGNED NOT NULL,
  `entry`     INT UNSIGNED NOT NULL,
  `map`       SMALLINT UNSIGNED NOT NULL DEFAULT 0,
  `x`         FLOAT NOT NULL,
  `y`         FLOAT NOT NULL,
  `z`         FLOAT NOT NULL,
  `o`         FLOAT NOT NULL DEFAULT 0,
  `scale`     FLOAT NOT NULL DEFAULT 1.0,
  PRIMARY KEY (`go_id`),
  CONSTRAINT `fk_pg_phase` FOREIGN KEY (`phase_id`)
    REFERENCES `custom_phases`(`phase_id`) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- Tabla de criaturas (NPCs) dentro de cada fase
CREATE TABLE IF NOT EXISTS `custom_phase_creatures` (
  `creature_id` BIGINT UNSIGNED NOT NULL AUTO_INCREMENT,
  `phase_id`    INT UNSIGNED NOT NULL,
  `entry`       INT UNSIGNED NOT NULL,
  `map`         SMALLINT UNSIGNED NOT NULL DEFAULT 0,
  `x`           FLOAT NOT NULL,
  `y`           FLOAT NOT NULL,
  `z`           FLOAT NOT NULL,
  `o`           FLOAT NOT NULL DEFAULT 0,
  PRIMARY KEY (`creature_id`),
  CONSTRAINT `fk_pc_phase` FOREIGN KEY (`phase_id`)
    REFERENCES `custom_phases`(`phase_id`) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- Tabla que guarda en qué fase está cada jugador al desconectarse
CREATE TABLE IF NOT EXISTS `custom_player_active_phase` (
  `player_guid` BIGINT UNSIGNED NOT NULL,
  `phase_id`    INT UNSIGNED NOT NULL,
  PRIMARY KEY (`player_guid`),
  CONSTRAINT `fk_pap_phase` FOREIGN KEY (`phase_id`)
    REFERENCES `custom_phases`(`phase_id`) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

Pulsa F9 para ejecutar todo el bloque.

💡

Igual que en el PASO 9: la línea USE custom; del principio ya selecciona la base de datos correcta para las 5 tablas que vienen detrás — no hace falta tocar el árbol de la izquierda, todo se ejecuta en orden con el mismo F9.
